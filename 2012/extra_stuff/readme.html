A) About The Theme - top

This theme is designed to be easy to use for either static HTML pages, or to convert for a content management system (CMS). The design is broken down to allow each section to be separated into a self-contained content block. These blocks are marked with comment tags to make finding them easier:

<!-- Name of Section or Area -->
	<div id="TheIdOfTheContainer" >
		<some HTML content />
	</div>

Layouts - sample-layout.html

Text - sample-text.html

B) HTML Structure - top

This theme is a fixed-width design with pre-configured styles for creating layouts with one, two or three columns.

The default content area is commented to make it easy to find. Look for the HTML comment with the text "Page Content".

<!-- Page Content -->

If you would like to edit the color, font, or style of any elements in one of these columns, you would do the following:

	.contentArea a {
		color: [color];
	}

If you find that your new style is not overriding, it is most likely because of a specificity problem. Scroll down in your CSS file and make sure that there isn't a similar style that has more weight.

I.E.

	#Wrapper .contentArea a {
		color: [color];
	}

To ensure that your new styles are applied, make sure that they carry enough "weight" and that there isn't a style lower in the CSS file that is being applied after yours.

The HTML code is indented to make it easier to edit. There are also comment markers as a guides to find the beginning and end of major sections such as the top, header, slide show, page content and footer. Each of these sections is labeled with the appropriate tags and has a corresponding ID tag of the main container for that section.

You should use the included sample files as templates for your own layouts. Be sure to backup a file before editing it, or create a copy and work from it.

C) CSS Files and Structure - top

There are several CSS files included with this theme, but only a few will be necessary for customization. The CSS files are all located in the "/www/css" folder. Within that folder there are two sub-folders, "source" and "skins".

The first folder, "source", contains the original CSS files with formatted code and comments. Only a few files are in this folder, primarily these are the original CSS files for plugins being used in the design which establish the foundation styles allowing the plugging to function but do not control appearance. If you do find a CSS file that is uncommented and you need the original, check this folder.

The second folder, "skins", houses the skin specific style sheet for each custom skin. If you create a custom skin, you will need to save the style sheet for it in this folder.

  1. default.css - This is the foundation of all the site design and all skin. It controls the layout and design structure of the theme. Making modifications to this file will usually effect all skins unless it is a color or image related change. The skin files extend default.css to apply custom styles. Before you edit this file you should consider making a custom skin style sheet or editing the one you are already using for your design.
  2. /skins/skin-#.css - These are the skin specific style sheets. For the purposes of the online demo, these files are named numerically in a format such as, "skin-##.css". If you plan to continue using the dynamic style switching functions that are included in the demo, you will need to continue naming your files this way. If you are going to use a single skin you can name the file however you like, and even save it in a different folder if you choose. Once you have the skin CSS file for you custom skin, be sure to add a link to it in all your HTML files in the <HEAD> section of each page. you may also wish to edit this file if you choose to customize the main menu.

    Your link should look something like this:
    <link rel="stylesheet" type="text/css" href="css/skins/skin-123.css" />

Skinning the Theme

Each color style is controlled by a custom CSS file. These are located in "/www/css/skins". If you want to keep the ability to use different colors and change them on the fly you will want to maintain this structure. The images for each skin are saved in a folder under "/www/images/skins/skin-123/" where "skin-123" is the name of your CSS file. To create a custom skin, duplicate on of the existing skin CSS files and edit it to match the desired styles. You can easily output the custom graphics for you skin using one of the "Theme Graphics - for skinning.PSD" files located in the "Design\Files for skinning" folded included with the theme.

After creating the skin CSS file and a matching images folder, save your images to this folder. Use the "Save for web..." feature of Photoshop and select "All user slices" to get the desired graphics. You can create a custom skin in a few minutes by modifying the background colors of the main shape layers of the supplied files. If you want to get more detailed and tweak the output you can edit the individual highlights and shadow features of the layers above the backgrounds. Play with the different layers to see what effect they have. The file was designed to allow easy and fast skinning.

After you have the new skin created you will either need to add a link (see above) to the HEAD tag of each HTML page, or if you plane to use the included skin switcher you should be ready to go. You can add a thumbnail graphic to the slider and it will show your new skin in the animation.

IMPORTANT: If you are going to create a custom skin or do any advanced editing of the HTML files, you will likely need to edit the JavaScript loading file, "js/source/onLoad.js" This file controls loading and executing of many dynamic JavaScript functions used in the theme. A minified version of this file, "onLoad.min.js", also exists in the main "js/" folder. To reduce file size you can replace all references in your HTML files pointing at, "js/source/onLoad.js", with the minimized version of the file, "js/onLoad.min.js". If you do edit the source file you can minimize it and update the ".min" version using an online JavaScript compressor, or you can leave the HTML files pointing to the source file, it will not effect the theme's functionality either way.

To create a single skin for your site

The best way to go about creating your own skin, is by starting from one of the pre-made styles that come with the theme. An easy way to do this would be as follows:

  1. First, create a copy of an existing style sheet from "/www/css/skins/skin-#.css", and increment the # to the next unused value.
  2. Create a duplicate of the same skin's images folder located in "/www/images/skins/skin-#"
  3. Give your new images folder the same name as the CSS file by incrementing the number to match.
  4. Open the "Theme Graphics - for skinning.psd" file in the "Design/Files for skinning" folder and modify the colors to create your desired graphics.
  5. Output the graphics using "Save for web..." to the new skin's images folder. Choose "All user created slices" as the option for output method when asked.
  6. Open the new CSS file for your skin. Do a find and replace for the path to your images folder, this would be something like, Find: "skin-5", Replace with: "Skin-6", to update all occurrences of the image folder path in the file. Save when finished.
  7. Add a link in your HTML files pointing to your new CSS skin file:
    <link rel="stylesheet" type="text/css" href="css/skins/skin-#.css" />
  8. Save your HTML files and load the page to preview the new skin.
  9. Open your skin CSS file and begin editing the individual style properties to match the graphics you have created for your new skin.
  10. Save, upload and enjoy!

F) Contact form - top

Contact form setup

The theme includes a working contact form. This form uses PHP and requires a server running PHP for it to work.

Adding Your Email Address

To add your own email address to the form, open the file "contact-send.php" and at the very top of the file replace "yourname@yourwebsite.com" with your own email address. Be sure not to delete the quotes, semi-colon or other code when making this change.

$YourEmailAddress = "yourname@yourwebsite.com";

Problems Sending Emails

If you experience trouble sending emails you should verify the following:

  1. Make sure the address entered in "contact-send.php" is correct.
  2. Verify that PHP is installed on the server your site is running on.
  3. If necessary contact your hosting provider to ensure they do not need to activate something.

Pages with Contact Forms

You must include the following code in the <HEAD> tag of any page with the contact form.

<!-- form validation scripts (for contact page) -->
<script src="js/jquery.validate.min.js" type="text/javascript"></script>
	
<!-- Contact form triggers and AJAX submit -->
<script src="js/contactForm.js" type="text/javascript"></script>

For and example of this review the included contact form sample page.
