6.470 Arctic Heist - Sample MySQL Problem

Background

Difficulty: Easy
50 points

After hacking into the lair's mainframe, you discover a large database listing all of the mastermind's various associates and their addresses and assets. These will be prime pickings for future heists! However, time is short and you don't have time to download the whole database. You need to perform some queries on the database to get some useful information.

Briefing

The database contains the following tables:

users

  • user_id INT 11
  • username VARCHAR 50
  • password VARCHAR 50
  • address_id INT 11 NULL
  • status INT 11 DEFAULT 0

addresses

  • address_id INT 11
  • street_line_1 VARCHAR 50
  • city VARCHAR 50
  • state VARCHAR 2
  • zipcode VARCHAR 5

assets

  • user_id
  • address_id
  • amount

Keep in mind that you have been given a (limited) dump of the entire database which you may use to check your solutions. Your solutions will be tested against the entire database.

Users without addresses are the most difficult to track down. You will not bother with them right now when there is lower hanging fruit, but you want to get a sense of how many these are. Write a SQL query to find the number of users that do not have an address.
